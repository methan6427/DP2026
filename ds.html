Chapter 1

Introduction to Data Science

What is Data Science?

Learning about the world from data using computation. It combines three activities:

Exploration

Identifying patterns using visualizations (e.g., heatmaps).

Inference

Drawing reliable conclusions using statistics (e.g., A/B testing).

Prediction

Making informed guesses using machine learning (e.g., house price prediction).

Data Science Disciplines

Math & Statistics

Descriptive stats, probability, inferential stats, normalization, ML model foundations (linear algebra, calculus).

Computer Science

Python, R, SQL; version control (GitHub); software development practices.

Domain Expertise

Understanding business context, validating results, communicating insights to stakeholders.

Data Science Workflow

Phase Description Key Points
1. Data Collection & Storage Gather from surveys, web traffic, transactions, social media Storage: cloud (AWS, Azure, GCP) or on-premises; SQL (structured) / NoSQL (unstructured)
2. ETL Extract → Transform → Load Extract from DBs/APIs, transform/clean, load into data warehouse
3. Data Cleaning & Preparation Make data analysis-ready Validate types, handle duplicates, missing values, inconsistencies, outliers
4. Exploration & Visualization (EDA) Understand data structure and patterns Histograms, box plots, scatter plots, bar charts, descriptive stats
5. Experimentation & Prediction Test hypotheses and forecast A/B testing, Time Series, Machine Learning

Data Cleaning Steps

  • Validate data types — ensure dates, numbers, and categoricals are in the right format.
  • Handle duplicates — remove or aggregate; sometimes sum is correct (sales data).
  • Missing values — remove rows/cols, or impute with mean/median/mode, or use regression imputation.
  • Inconsistencies — "NYC" vs "New York City", "kg" vs "lbs", capitalization differences.
  • Outliers — detect via IQR or Z-score; decide to remove, transform, or keep.
📌 Outlier DetectionValues below Q1 − 1.5×IQR or above Q3 + 1.5×IQR are considered outliers.

Data Science Roles

Role Focus Tools
Data Engineer Build & maintain data infrastructure, ETL pipelines SQL, NoSQL, Python/Java/Scala, Hadoop, Spark, AWS/Azure
Data Analyst Transform data into insights, create dashboards SQL, Excel, Power BI/Tableau, Python/R
Data Scientist EDA, build & deploy ML models, solve complex problems Python (Pandas, NumPy, Scikit-learn), SQL
ML Scientist Specialize in ML modeling & deployment, deep learning TensorFlow, Keras, PyTorch, AWS/Azure cloud

Machine Learning Types

Supervised Learning

Uses labeled data. Trains a model to predict outcomes (classification/regression). Example: customer churn prediction.

Unsupervised Learning

No labels. Finds patterns: clustering (K-means), association, dimensionality reduction. Example: customer segmentation (RFM).

Reinforcement Learning

Sequential decision-making with reward/penalty. Example: game-playing robots.

🎯 Model Evaluation Metrics
  • Accuracy: proportion of correct predictions
  • Precision: of predicted positives, how many are truly positive
  • Recall: of actual positives, how many were correctly identified
  • F1-Score: harmonic mean of precision and recall

NumPy

Why NumPy? Stores homogeneous data densely in memory; supports element-wise (vectorized) operations without loops — much faster than Python lists for math.
import numpy as np

# Create arrays
a1 = np.array([1, 2, 3, 4, 5])               # 1-D
a2 = np.array([[1,2,3],[4,5,6]])              # 2-D
a3 = np.array([[[1,2],[3,4]],[[5,6],[7,8]]]) # 3-D

# Array properties
a2.ndim    # number of dimensions: 2
a2.shape   # (2, 3)

# Indexing: array[row, col]
a2[0, 2]   # row 0, col 2 → 3

# Slicing: array[row_start:row_end, col_start:col_end]
a2[:2, 1:] # rows 0-1, cols 1+ → shape (2,2)

# Element-wise operations (no loop needed!)
bmi = weight / height**2

# Special arrays
np.zeros((3,3))
np.ones((2,4))
np.arange(0, 10, 2)

# Reshape & transpose
a2.reshape(3, 2)   # same data, new shape
a2.T               # transpose — returns new array

# Aggregation
np.sum(a2), np.mean(a2), np.max(a2), np.min(a2)

# Broadcasting: operate on different-shaped arrays
a2 + 10  # adds 10 to every element
⚠️ NumPy Slices are VIEWSUnlike Python lists, NumPy slices point to the same memory. Modifying a slice modifies the original. Use .copy() to avoid this.

Statistics for Data Science

Types of Statistics & Data

Descriptive

Summarize what happened in this group (mean, median, charts). No inference beyond the data at hand.

Inferential

Make predictions about a population from a sample. Hypothesis testing, confidence intervals.

Data Type Subtypes Examples
Numeric (Quantitative) Continuous Height, time, temperature, battery %
Discrete Number of students, steps per day, siblings
Categorical (Qualitative) Nominal (unordered) Marital status, country, blood type
Ordinal (ordered) Education level, satisfaction rating

Central Tendency Measures

Measure Formula When to use Weakness
Mean (Arithmetic) Σx / n Symmetric data, no extreme outliers Highly sensitive to outliers
Median Middle value of sorted data Skewed data or when outliers exist Ignores actual values
Mode Most frequent value Categorical data, finding most common item May not exist or be multiple
Geometric Mean ⁿ√(x₁·x₂···xₙ) Multiplicative growth (investment returns) Cannot include 0 or negatives
Harmonic Mean n / Σ(1/xᵢ) Rates (average speed) Distorted by small values

Measures of Spread

Range

Range = max − min

Simple but misleading with outliers; ignores how data is distributed between extremes.

Variance

Population: σ² = Σ(xᵢ − μ)² / n   |   Sample: s² = Σ(xᵢ − x̄)² / (n−1)

Average squared deviation from mean. Unit is squared — hard to interpret directly.

Standard Deviation

σ = √variance

Same unit as data. Small std = data close to mean; large std = data spread out.

Z-Score

z = (x − μ) / σ

How many standard deviations a point is from the mean. Useful for comparing across datasets.

Quartiles & IQR

Q1 (25th pct)

25% of data falls below this value.

Q2 / Median (50th)

Middle of the data.

Q3 (75th pct)

75% of data falls below this value.

IQR = Q3 − Q1

Middle 50% of data. Robust to outliers.

📦 Boxplot ComponentsBox = IQR (Q1 to Q3). Line inside = Median (Q2). Whiskers extend to 1.5×IQR above Q3 / below Q1. Points outside whiskers = outliers.

Distribution Shapes

Shape Mean vs Median Description
Symmetric (Normal) Mean ≈ Median ≈ Mode Bell curve; data evenly spread around center
Left (Negative) Skewed Mean < Median Long tail on the left; most data concentrated on right
Right (Positive) Skewed Mean > Median Long tail on the right; most data concentrated on left (e.g., income)
Normal Distribution — Empirical Rule (68-95-99.7)
  • 68% of data within 1σ of mean
  • 95% within 2σ
  • 99.7% within 3σ

Sampling

Term Definition
Population All possible cases meeting the criteria
Sample A subset of the population used for analysis
Parameter A metric calculated from the population
Statistic A metric calculated from the sample
Sampling Error Difference between population parameter and sample statistic
Method How Pros Cons
Simple Random Sampling (SRS) Each member has equal chance of selection Easy; representative for homogeneous populations May miss small subgroups
Stratified Sampling Divide into strata; random sample from each Ensures all subgroups represented; lower sampling error Complex; requires knowledge of strata
Systematic Every kth element Simple Can introduce bias if pattern exists
Cluster Divide into clusters; randomly select clusters Efficient for geographically dispersed populations Higher sampling error 
# SRS
sample = df.sample(n=200)

# Stratified by nationality
stratified = df.groupby('Nationality').sample(n=1)

# Sampling error
error = df['score'].mean() - sample['score'].mean()
✅ Stratified vs SRSStratified sampling gives lower sampling error when subgroups exist in the population, because it guarantees representation of every group.

Combining Datasets with Pandas

Types of Joins

🔵 Inner Join

Returns only rows that have a match in both tables. Unmatched rows are excluded.

pd.merge(df1, df2, on='ID', how='inner')

🟢 Left Join

All rows from the left table + matched rows from right. Unmatched right → NaN.

pd.merge(df1, df2, on='ID', how='left')

🟠 Right Join

All rows from the right table + matched rows from left. Unmatched left → NaN.

pd.merge(df1, df2, on='ID', how='right')

🟣 Outer Join

All rows from both tables. NaN wherever no match exists in either side.

pd.merge(df1, df2, on='ID', how='outer')
📌 indicator=TrueAdd indicator=True to the merge to get a _merge column showing whether each row came from 'left_only', 'right_only', or 'both'.

Which join to choose?

Scenario Use
Only want records that exist in both datasets Inner
Want to keep all records from the primary (left) dataset Left
Want to keep all records from a secondary (right) dataset Right
Want a complete picture of all records from both datasets Outer

Suffixes for Overlapping Columns

When both DataFrames have same-named columns (other than the key), use suffixes:

merged = pd.merge(wb, gaza, on="commodity", how="inner",
                  suffixes=("_wb", "_gaza"))
# Columns: price_wb, price_gaza
merged[merged["price_wb"] > merged["price_gaza"]]

Concatenation with pd.concat()

Vertical (axis=0)

Stack rows on top of each other. DataFrames must have same columns. Use ignore_index=True to reset index.

Horizontal (axis=1)

Add columns side-by-side. Aligns on index. Missing rows fill with NaN.

# Vertical — combine Q1 and Q2 sales
annual = pd.concat([q1, q2], axis=0, ignore_index=True)

# Horizontal — add profit columns to sales
combined = pd.concat([sales, profit], axis=1)
⚠️ Watch out for
  • Different indices with axis=0: unmatched rows get NaN
  • Duplicate column names with axis=1: concat adds _x/_y suffixes
  • Missing columns: NaN fills those cells

Full Syntax Reference

# pd.merge
pd.merge(left, right,
         on='key',           # shared column name
         how='inner',        # inner/left/right/outer
         suffixes=('_x','_y'), # for overlapping cols
         indicator=True)      # adds _merge column

# pd.concat
pd.concat([df1, df2],
          axis=0,             # 0=rows, 1=cols
          ignore_index=True) # reset index
